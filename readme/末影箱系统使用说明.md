# 末影箱系统使用说明

## 📋 系统概述

末影箱系统已经完成，实现了以下功能：
1. **每层结束时自动打开** - 击败Boss后自动触发
2. **随机生成作物选项** - 根据层数生成3-5个可选作物
3. **代价系统** - 支持金币、生命值、卡牌三种代价类型
4. **自动种植** - 选择作物后自动种植到空余地块

---

## 🎮 使用方法

### 1. 打开末影箱

系统会在每层完成时自动打开，也可以通过代码手动打开：

```csharp
var gameManager = GameRoot.Instance.GameManager;
var chestData = gameManager.OpenEnderChest(floorNumber);

if (chestData != null)
{
    GD.Print($"末影箱已打开，提供 {chestData.AvailableCrops.Count} 个选项");
}
```

### 2. 查看可用作物

```csharp
var chestData = gameManager.GetCurrentEnderChest();
if (chestData != null)
{
    foreach (var option in chestData.AvailableCrops)
    {
        GD.Print($"作物: {option.CropData.Name}");
        GD.Print($"代价: {option.Cost.Description}");
        GD.Print($"可用: {option.IsAvailable}");
    }
}
```

### 3. 选择作物

```csharp
bool success = gameManager.SelectCropFromEnderChest("crop_health_boost");
if (success)
{
    GD.Print("成功获得作物并已种植到农场");
}
else
{
    GD.Print("选择失败（可能是资源不足或没有空余地块）");
}
```

## 💰 代价系统

### 代价类型

1. **金币代价 (Gold)**
   - 最常见（70%概率）
   - 基础价格：20金币 × 时间因子 × 效果因子 × 层数因子
   - 示例：生命之果可能需要30-50金币

2. **生命值代价 (Health)**
   - 中等概率（20%概率）
   - 代价 = 金币代价的10%
   - 至少保留1点生命值

3. **卡牌代价 (Card)**
   - 较少概率（10%概率）
   - 删除1张随机卡牌
   - 需要卡组中至少还有卡牌

### 代价计算规则

```csharp
基础代价 = 20 × 时间因子 × 效果因子 × 层数因子

时间因子 = 生长时间 / 300秒（5分钟为基准）
效果因子：
  - 属性加成：1.5
  - 诅咒权衡：1.2
  - 遗忘：0.8
层数因子 = 1.0 + (层数 - 1) × 0.1
```

---

## 🌾 作物生成规则

### 数量规则
- 基础数量：3个选项
- 每3层增加1个选项
- 最多5个选项

### 稀有度规则
- 普通作物：所有层数都可能出现
- 珍稀作物（如母体核心）：10层以上才可能出现

### 随机性
- 从所有可用作物中随机选择
- 不会重复选择相同作物

## 🔧 系统架构

### 核心类

1. **EnderChestSystem** (`Scripts/Managers/EnderChestSystem.cs`)
   - 管理末影箱的打开和选择逻辑
   - 生成作物选项和代价
   - 处理代价支付

2. **EnderChestData** (`Scripts/DataTypes/EnderChestData.cs`)
   - 存储末影箱数据
   - 包含可用作物列表和状态

3. **EnderChestCost** (`Scripts/DataTypes/EnderChestData.cs`)
   - 定义代价类型和数值
   - 支持多种代价类型

### 事件系统

系统通过EventBus发送以下事件：
- `EnderChestOpened` - 末影箱打开时触发
- `CropSelectedFromChest` - 选择作物时触发

### 集成点

- **GameRoot**: 注册了EnderChestSystem和FarmingSystem单例
- **GameManager**: 提供了便捷的接口方法
- **MapSystem**: 在CompleteFloor时触发FloorCompleted事件
- **EnderChestSystem**: 自动订阅FloorCompleted事件

## 💡 使用建议

### 在UI中集成

当末影箱打开时，显示选择界面：

```csharp
// 订阅事件
eventBus.EnderChestOpened += OnEnderChestOpened;

private void OnEnderChestOpened(EnderChestData chestData)
{
    // 显示末影箱UI
    ShowEnderChestUI(chestData);
}

private void ShowEnderChestUI(EnderChestData chestData)
{
    foreach (var option in chestData.AvailableCrops)
    {
        // 创建UI元素显示作物信息
        CreateCropOptionUI(option, (cropId) => {
            // 玩家点击选择
            if (gameManager.SelectCropFromEnderChest(cropId))
            {
                // 选择成功，关闭UI
                HideEnderChestUI();
            }
            else
            {
                // 选择失败，显示错误提示
                ShowError("无法获得该作物（资源不足或没有空余地块）");
            }
        });
    }
}
```

### 代价显示

在UI中清晰地显示代价：

```csharp
string GetCostDisplayText(EnderChestCost cost)
{
    switch (cost.Type)
    {
        case EnderChestCostType.Gold:
            return $"💰 {cost.Value} 金币";
        case EnderChestCostType.Health:
            return $"❤️ {cost.Value} 生命值";
        case EnderChestCostType.Card:
            return $"🃏 删除1张卡牌";
        default:
            return cost.Description;
    }
}
```

### 可用性检查

在显示作物选项时，根据可用性改变UI状态：

```csharp
if (!option.IsAvailable)
{
    // 灰色显示，禁用按钮
    cropButton.Disabled = true;
    cropButton.Modulate = new Color(0.5f, 0.5f, 0.5f);
    // 显示原因
    reasonLabel.Text = "资源不足";
}
```

## 🐛 注意事项

1. **地块检查**: 系统会自动查找空余地块，如果没有空余地块，选择会失败
2. **资源检查**: 在选择前会检查玩家是否有足够资源支付代价
3. **自动种植**: 选择成功后，作物会自动种植到第一个空余地块
4. **事件触发**: 选择作物后会触发CropPlanted事件，农场系统会自动处理

## 📝 示例流程

1. 玩家击败Boss，完成第1层
2. MapSystem触发FloorCompleted事件
3. EnderChestSystem自动打开末影箱
4. 系统生成3个随机作物选项，每个都有随机代价
5. UI显示末影箱界面，展示所有选项
6. 玩家选择"生命之果"，需要支付35金币
7. 系统扣除35金币，将作物种植到空余地块
8. 触发CropPlanted事件，农场系统开始管理作物生长
