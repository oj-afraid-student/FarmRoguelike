# 保存系统使用说明

## 📋 系统概述

SaveManager 已经完成，实现了完整的数据持久化功能：
1. **游戏数据保存** - 保存玩家数据、游戏进度、作物状态等
2. **游戏数据加载** - 恢复所有游戏状态
3. **快速保存/加载** - 便捷的快速存档功能
4. **离线生长计算** - 加载时自动计算离线期间的作物生长

---

## 🎮 使用方法

### 1. 保存游戏

```csharp
var saveManager = GameRoot.Instance.SaveManager;

// 普通保存
bool success = saveManager.SaveGame();
if (success)
{
    GD.Print("游戏已保存");
}

// 快速保存
bool quickSuccess = saveManager.QuickSave();
```

### 2. 加载游戏

```csharp
var saveManager = GameRoot.Instance.SaveManager;

// 普通加载
bool success = saveManager.LoadGame();
if (success)
{
    GD.Print("游戏已加载");
}

// 快速加载
bool quickSuccess = saveManager.QuickLoad();
```

### 3. 检查保存文件

```csharp
var saveManager = GameRoot.Instance.SaveManager;

// 检查是否有保存文件
if (saveManager.HasSaveFile())
{
    GD.Print("存在保存文件");
    
    // 获取保存文件信息
    var info = saveManager.GetSaveFileInfo();
    if (info != null)
    {
        GD.Print($"保存时间: {info.SaveTime}");
        GD.Print($"当前层数: {info.Floor}");
    }
}
```

### 4. 删除保存文件

```csharp
var saveManager = GameRoot.Instance.SaveManager;

// 删除保存文件
bool deleted = saveManager.DeleteSaveFile();
if (deleted)
{
    GD.Print("保存文件已删除");
}
```

## 💾 保存的数据

### 玩家数据
- 生命值、攻击力、防御力等属性
- 金币数量
- 卡组
- 物品库存

### 游戏进度
- 当前层数
- 游戏状态

### 作物系统数据
- 所有地块的作物状态
- 种植时间
- 生长进度
- 最后更新时间（用于计算离线生长）

### 作物效果数据
- 所有激活的作物效果
- 效果应用时间

### 战斗物资数据
- 玩家拥有的所有物资
- 物资数量

## 🔄 离线生长计算

系统会在加载存档时自动计算离线期间的作物生长：

```csharp
// 保存时记录时间
FarmingSaveData.LastUpdateTime = DateTime.Now;

// 加载时计算离线时间
var offlineTime = (DateTime.Now - LastUpdateTime).TotalSeconds;

// 自动更新作物生长进度
plot.GrowthProgress = Mathf.Min(1.0f, elapsedTime / cropData.GrowthTimeSeconds);
```

## 📁 文件位置

保存文件存储在：
- **Windows**: `%APPDATA%/Godot/app_userdata/[项目名]/saves/`
- **Linux**: `~/.local/share/godot/app_userdata/[项目名]/saves/`
- **macOS**: `~/Library/Application Support/Godot/app_userdata/[项目名]/saves/`

文件名称：
- `save_game.json` - 普通保存
- `quick_save.json` - 快速保存

## 🔧 系统架构

### 核心类

1. **SaveManager** (`Scripts/Managers/SaveManager.cs`)
   - 管理保存和加载
   - 处理JSON序列化
   - 文件操作

2. **GameSaveData** (`Scripts/DataTypes/SaveData.cs`)
   - 保存数据结构
   - 包含所有需要保存的数据

3. **FarmingSaveData** (`Scripts/DataTypes/SaveData.cs`)
   - 农场数据
   - 地块数据

4. **CropEffectSaveData** (`Scripts/DataTypes/SaveData.cs`)
   - 作物效果数据

5. **CombatResourceSaveData** (`Scripts/DataTypes/SaveData.cs`)
   - 物资数据

### 数据恢复方法

- `FarmingSystem.RestorePlots()` - 恢复地块数据
- `CropEffectSystem.ClearAllEffects()` - 清除效果
- `CombatResourceSystem.AddResource()` - 恢复物资

## 💡 使用建议

### 在UI中集成

```csharp
// 保存按钮
void OnSaveButtonPressed()
{
    var saveManager = GameRoot.Instance.SaveManager;
    if (saveManager.SaveGame())
    {
        ShowNotification("游戏已保存");
    }
    else
    {
        ShowNotification("保存失败");
    }
}

// 加载按钮
void OnLoadButtonPressed()
{
    var saveManager = GameRoot.Instance.SaveManager;
    if (saveManager.HasSaveFile())
    {
        // 显示确认对话框
        ShowConfirmDialog("确定要加载存档吗？当前进度将丢失。", () => {
            if (saveManager.LoadGame())
            {
                ShowNotification("游戏已加载");
                // 刷新UI
                RefreshAllUI();
            }
        });
    }
    else
    {
        ShowNotification("没有保存文件");
    }
}
```

### 自动保存

可以在关键节点自动保存：

```csharp
// 在GameManager中
private void OnFloorCompleted(int floorNumber)
{
    // 每完成一层自动保存
    GameRoot.Instance.SaveManager.QuickSave();
}
```

## 🐛 注意事项

1. **JSON序列化**: 使用System.Text.Json，确保所有数据类标记为[Serializable]
2. **文件路径**: 使用Godot的user://路径，确保跨平台兼容
3. **数据验证**: 加载时验证数据完整性
4. **错误处理**: 保存/加载失败时给出明确提示
5. **版本控制**: 保存数据包含版本号，便于未来升级
