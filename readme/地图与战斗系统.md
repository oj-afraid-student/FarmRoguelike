# 地图与战斗系统运行逻辑

本文档详细说明了游戏内主要的进度驱动模块：**地图探索（MapSystem）**和**战斗交锋（CombatSystem）**的底层运行逻辑。这两者相互依存，并通过 `GameManager` 与 `EventBus` 进行状态调度。

---

## 1. 核心流转架构 (GameManager 中心调度)

整个游戏的状态机由 `GameManager.cs` 控制，分为：
- `MainMenu`（主菜单） 
- `MapExploration`（地图探索模式）
- `Combat`（战斗模式）
- `Farm`（农场模式 - 当前版本未作为主干进度）
- `GameOver`（游戏结束）

### 1.1 状态流转时序图
1. 游戏通过 `GameManager.StartNewGame()` 启动，生成初始层数（CurrentFloor=1），并触发状态变为 `MapExploration`。
2. 地图UI构建网格，玩家点击移动按钮后，`MapSystem` 判定合法性并触发 `EventBus.RoomEntered` 事件。
3. `GameManager` 捕获 `RoomEntered` 事件：
   - 遇到 **Reward (奖励) / Trap (陷阱)** ：处理对应数据的增减，发送 `NotificationRequested` 给UI弹窗显示，随后保留在探索模式。
   - 遇到 **Combat / Boss** ：从 `DataManager` 动态抽取随机敌人数据，并切换状态至 `Combat` 模式。
4. 在 `Combat` 状态下完成回合制战斗，当一方生命值归零时发出 `CombatEnded` 信号。
5. `GameManager` 收到胜利信号后，将状态重置回 `MapExploration`，允许玩家继续寻找下一个房间；如果失败，则切入 `GameOver`。

## 2. 地图探索系统 (MapSystem)

### 2.1 网格生成逻辑
进入新的楼层时，`MapSystem.GenerateFloor(int floor)` 被调用。地图大小和层数线性正相关（如 $Size = 3 + Floor \div 3$）。
生成规则包括：
1. 固定起点 `(0,0)` 为安全/奖励点。
2. 依据尺寸，将对角最远端设定为 Boss点。
3. 中间空白区域由随机算法依据设定好的权重表（Combat: 60%, Reward: 20%, Trap: 20% 等）填充不同类型的方块数据 `RoomData`。

### 2.2 移动与探索
玩家在地图界面的上下左右按钮会调用 `MapSystem.GetAvailableMoves()` 进行计算，确保只在生成的网格范围和尚未探索的相邻区域内行动。
当进入某个房间后，底层数据会标记 `room.IsCleared = true`。如果后续走回头路，`GameManager` 会判断其是否清理过，若已被清理则只发弹窗告知安全，不再触发事件。

## 3. 回合制战斗系统 (CombatSystem)

负责维护玩家与特定敌人交锋状态的数据层与规则层。

### 3.1 战斗初始化序列
一旦 `GameManager` 获取随机怪物 `EnemyData` 并发送 `EventBus.EmitCombatStarted(enemyId)`，`CombatSystem.StartCombat()` 被激活。
1. 重置双方血量缓存器：提取玩家护甲（`_playerDefenseThisTurn = 0`）和 Action Points（3点）。
2. **初始化卡组：** 将玩家所有已知卡牌拷贝入 `_playerDrawPile` 并打乱洗牌。
3. **初始抽卡：** 保证首回合抽到 4 张卡牌进入 `_playerHand`。
4. **属性加载：** 根据传来的 `EnemyData` 初始化当前关底敌人的攻击与血量。

### 3.2 卡牌结算逻辑 (Player Turn)
玩家每一次操作手卡都会发送 `CardPlayed` 事件：
1. 检查当前 AP 是否足够。扣减 AP 点数。
2. 将卡牌从 `_playerHand` 中移出，放入 `_playerDiscardPile`（弃牌堆）。
3. **计算 Effects：**
   通过比对被调用的卡牌ID，系统会解析外部加载带来的卡牌 JSON `Effects`：
   - `"damage": 8` -> 将获取玩家基础攻击叠加后，再扣减该回合敌方的额外防御，算出纯伤害。
   - `"defense": 5` -> 增加本回合护甲减免数值 `_playerDefenseThisTurn`。
   - `"weakness": 3` -> 将暂时扣除敌人的基础攻击力 `_currentEnemyAttack` 以降低其对玩家的威胁。

### 3.3 回合结束设计 (End Turn)
玩家可通过 UI 点击结束回合（优先触发 `CombatSystem.EndPlayerTurnEarly()`），此阶段会发生：
1. 把残余在手里的多余卡牌全数丢入弃牌堆（清空 `_playerHand`）。
2. 让出行动权利给到电脑AI，切换到 `EnemyTurn`。

### 3.4 敌人AI运转与伤害结算 (Enemy Turn)
敌人进行行动受其专属字段 `AiPattern = ["Attack", "Defend", "Debuff"]` 的轮询控制：
1. **获取当前行为：** 提取 `AiPattern[_enemyAiStateIndex % AiPattern.Count]` 并累加索引以指向下一轮。
2. **多态行为执行：** 
   - `"Attack"` 行为将用它的剩余攻击力硬碰硬计算玩家当前所带的护甲。溢出的伤害由 `GameManager.ApplyDamageToPlayer()` 处理真实扣血。
   - `"Defend"` 则让怪在此回合自我附加 `_currentEnemyDefense`。
3. **回合重置与再次抽牌：** 攻击完成后，敌人回合闭幕。玩家本回合堆积的护甲将全部重置归 `0`，紧接着系统从牌库顶部再度抽取 4 张派发给玩家并开放下个回合交互。

## 4. UI 渲染表现层 (UIManager)

该层级只负责**“渲染从系统拿到的纯数据集”**以及**“拦截用户的点击下发系统指令”**，不包含游戏逻辑判断。

- **地图界面同步：** 监听 `EventBus.RoomEntered` 并提取 `CurrentRoom` 坐标计算，只保留四周可合法通行的上下左右按钮。如果某行某列无路可走则不渲染按钮。
- **战斗手牌同步：** 订阅 `CardPlayed` 和 `TurnEnded` 等事件重清/重绘屏幕上的手牌排列。
- **伤害飞字与弹窗：** 通过在全局 `CanvasLayer` 之上实例化文本框或颜色闪烁面板展现被抽像化剥离出的诸如"获得20金币"、"被减攻"的通知气泡。
