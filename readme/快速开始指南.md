# 快速开始指南

## 🚀 5分钟快速上手

### 第一步：验证系统（1分钟）

在Godot编辑器中运行游戏，检查控制台输出：
```
✓ GameRoot initialized
✓ EventBus 初始化完成
✓ FarmingSystem 初始化完成
✓ CropEffectSystem 初始化完成
✓ EnderChestSystem 初始化完成
✓ CombatResourceSystem 初始化完成
```

如果看到这些输出，说明所有系统已正确初始化。

### 第二步：测试核心功能（2分钟）

在场景中创建 `SystemTestHelper` 节点：

1. 在场景树中右键 → 添加节点
2. 选择 `Node` 类型
3. 命名为 `SystemTestHelper`
4. 在检查器中添加脚本：`Scripts/Testing/SystemTestHelper.cs`

运行以下代码测试：
```csharp
// 在某个脚本中
var testHelper = GetNode<SystemTestHelper>("SystemTestHelper");
testHelper.RunAllTests();
```

### 第三步：创建UI场景（2分钟）

#### 快速创建FarmUI

1. **创建场景**
   - 场景 → 新建场景
   - 添加 `Control` 节点，命名为 `FarmUI`
   - 保存为 `UI/FarmUI.tscn`

2. **绑定脚本**
   - 选择 `FarmUI` 节点
   - 在检查器中点击"脚本"
   - 选择 `Scripts/UI/FarmUIController.cs`

3. **创建基本UI**
   - 添加 `VBoxContainer` 作为主容器
   - 添加 `GridContainer`，命名为 `PlotsContainer`（3列）
   - 添加 `VBoxContainer`，命名为 `CropSelectionContainer`
   - 添加 `VBoxContainer`，命名为 `ActiveEffectsContainer`
   - 添加 `VBoxContainer`，命名为 `ResourcesContainer`
   - 添加 `Button`，命名为 `CloseButton`

4. **连接Export变量**
   - 在检查器的脚本部分
   - 将对应的节点拖拽到Export变量中

#### 快速创建EnderChestUI

1. **创建场景**
   - 场景 → 新建场景
   - 添加 `Control` 节点，命名为 `EnderChestUI`
   - 保存为 `UI/EnderChestUI.tscn`

2. **绑定脚本**
   - 选择 `EnderChestUI` 节点
   - 绑定 `Scripts/UI/EnderChestUIController.cs`

3. **创建基本UI**
   - 添加 `VBoxContainer` 作为主容器
   - 添加 `Label`，命名为 `TitleLabel`
   - 添加 `Label`，命名为 `DescriptionLabel`
   - 添加 `VBoxContainer`，命名为 `CropsContainer`
   - 添加 `HBoxContainer`，命名为 `PlayerInfo`
     - 添加 `Label`，命名为 `PlayerGoldLabel`
     - 添加 `Label`，命名为 `PlayerHealthLabel`
   - 添加 `Button`，命名为 `CloseButton`

4. **连接Export变量**
   - 将所有节点连接到对应的Export变量

## 🐛 常见问题快速解决

### Q1: 系统未初始化
**解决**: 检查GameRoot是否正确添加到场景树

### Q2: UI不显示
**解决**: 
1. 检查UI场景是否正确加载
2. 检查Export变量是否正确连接
3. 检查节点是否可见

### Q3: 事件不触发
**解决**:
1. 检查EventBus是否正确初始化
2. 检查事件订阅是否在_Ready()中完成

### Q4: 数据未保存
**解决**:
1. 检查SaveManager是否在GameRoot中注册
2. 检查文件权限
