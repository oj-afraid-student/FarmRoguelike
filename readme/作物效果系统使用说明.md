# 作物效果系统使用说明

## 📋 系统概述

作物效果系统已经完成，支持三种效果类型：
1. **属性加成 (StatBoost)** - 永久提升玩家属性
2. **诅咒权衡 (CurseTrade)** - 降低某些属性，提升其他属性
3. **遗忘 (Forget)** - 删除卡组中的卡牌

---

## 🎮 使用方法

### 1. 应用作物效果

```csharp
// 通过GameManager应用效果
var gameManager = GameRoot.Instance.GameManager;
bool success = gameManager.ApplyCropEffect("crop_health_boost");

if (success)
{
    GD.Print("生命之果效果已应用！");
}
```

### 2. 检查效果是否已激活

```csharp
bool isActive = gameManager.IsCropEffectActive("crop_health_boost");
```

### 3. 获取所有激活的效果

```csharp
var activeEffects = gameManager.GetActiveCropEffects();
foreach (var effect in activeEffects)
{
    GD.Print($"激活效果: {effect.CropName} ({effect.EffectType})");
}
```

### 4. 移除作物效果

```csharp
gameManager.RemoveCropEffect("crop_health_boost");
```

## 🔧 系统架构

### 核心类

1. **CropEffectSystem** (`Scripts/Managers/CropEffectSystem.cs`)
   - 管理所有激活的作物效果
   - 处理效果的应用和移除
   - 与GameManager和PlayerData交互

2. **CropData** (`Scripts/DataTypes/GameData.cs`)
   - 扩展了 `EffectType` 和 `EffectData` 字段
   - 支持三种效果类型的数据定义

3. **ActiveCropEffect** (`Scripts/Managers/CropEffectSystem.cs`)
   - 记录已激活的效果信息
   - 包含作物ID、名称、效果类型、应用时间

### 事件系统

系统通过EventBus发送以下事件：
- `CropEffectApplied` - 效果应用时触发
- `CropEffectRemoved` - 效果移除时触发

### 集成点

- **GameRoot**: 注册了CropEffectSystem单例
- **GameManager**: 提供了便捷的接口方法
- **DataManager**: 包含了所有作物数据定义

## 💡 使用建议

### 在UI中集成

当玩家收获作物时，可以显示一个对话框询问是否应用效果：

```csharp
// 在收获作物后
var cropData = dataManager.GetCrop(cropId);
if (cropData.EffectType != GameEnums.CropEffectType.StatBoost || 
    cropData.EffectData == null)
{
    // 无效果，直接收获
    return;
}

// 显示确认对话框
ShowEffectDialog(cropData, () => {
    gameManager.ApplyCropEffect(cropId);
});
```

### 在末影箱系统中使用

当玩家从末影箱选择作物时，可以选择是否立即应用效果：

```csharp
// 玩家选择作物后
void OnCropSelected(string cropId)
{
    // 先添加到农场种植
    farmingSystem.TryPlantCrop(plotIndex, cropId);
    
    // 询问是否立即应用效果（如果作物有效果）
    var cropData = dataManager.GetCrop(cropId);
    if (cropData.EffectData != null)
    {
        ShowApplyEffectOption(cropData);
    }
}
```

## 🐛 注意事项

1. **效果叠加**: 目前系统允许同一作物的效果多次应用（如果收获多次），但实际游戏中可能需要限制
2. **效果移除**: 移除效果时不会恢复原始属性值，需要根据游戏设计决定是否需要恢复机制
3. **数据持久化**: 激活的效果需要在保存系统中保存，以便游戏重启后恢复
4. **UI反馈**: 应用效果后应该给玩家明确的视觉反馈，显示属性变化
